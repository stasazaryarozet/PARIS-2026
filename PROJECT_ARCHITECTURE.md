## КОНЦЕПЦИЯ: ZOOM IN / ZOOM OUT

Проект существует в иерархической структуре с разными уровнями масштаба:

```
Дел Вообще (все проекты, все сферы)
  ↓
Ольга (одна из сфер)
  ↓
Дизайн-путешествия (категория проектов Ольги)
  ↓
Париж 2026 (конкретный проект) ← МЫ ЗДЕСЬ (Zoom In)
```

**Zoom In**: Углубление в конкретный проект (Париж 2026)  
**Zoom Out**: Выход на уровень выше (Дизайн-путешествия → Ольга → Дел Вообще)

## РАБОТА С УРОВНЯМИ

### Zoom In (углубление)

**Сценарий**: Работа над конкретным проектом

1. Открыть Cursor в папке конкретного проекта (`PARIS 2026/`)
2. Все изменения делаются в этой папке
3. Контекст ограничен этим проектом
4. Фокус на конкретной задаче

**Пример**: Работа над сайтом Париж 2026 → все файлы в `PARIS 2026/`

---

### Zoom Out (выход на уровень выше)

**Сценарий**: Нужен общий контекст или работа с несколькими проектами

1. Переключиться на папку уровня выше (`Дизайн-путешествия/` или `Ольга/`)
2. Видеть несколько проектов одновременно
3. Работать с общими ресурсами (SHARED/)
4. Координировать между проектами

**Пример**: Обновить общий список адресатов для всех туров → `Дизайн-путешествия/SHARED/contacts/`

## СВЯЗИ МЕЖДУ УРОВНЯМИ

### Общие ресурсы

**Где хранить общие вещи** (списки контактов, шаблоны, ресурсы)?

**Принцип**:
- **Контакты** → на уровне Ольги (`Ольга/Контакты/`) — используются всеми проектами Ольги
- **Ресурсы туров** → на уровне категории (`Дизайн-путешествия/SHARED/`) — используются только турами

**Почему**:
- Контакты нужны всем проектам Ольги (не только турам)
- Ресурсы туров специфичны для категории "Дизайн-путешествия"

**Пример**:
```
Ольга/Контакты/ADDRESS_LIST.csv
  → Используется в: все проекты Ольги (туры и другие)

Дизайн-путешествия/SHARED/templates/
  → Используется в: только туры (Париж 2026, будущие туры)
```

### Изоляция проектов

**Принцип**: Каждый проект изолирован в своей папке

**Почему**: Проект может быть завершен, заархивирован, или удален без влияния на другие

**Пример**:
```
Дизайн-путешествия/
├── Париж 2026/ (активный)
├── Сентябрь 2025/ (архив, не трогаем)
└── Будущие проекты/ (планирование)
```

**Текущая рабочая папка**: `PARIS 2026/`

## ПРИНЦИПЫ АРХИТЕКТУРЫ

### q1: Можно ли без этого обойтись?

**Иерархия нужна**:
- Масштабирование (один проект → много проектов)
- Изоляция (завершенный проект не мешает новому)
- Общие ресурсы (не дублировать для каждого проекта)
- Контекст (понимать, где находишься)

**Минимум уровней**: 4 уровня достаточно (больше = избыточность)

### Изоляция

Каждый проект — отдельная папка, отдельный Git репозиторий (если нужно)

**Преимущества**:
- Проект можно заархивировать/удалить
- Не влияет на другие проекты
- Можно отдать проект другому разработчику

### Общие ресурсы

Общее хранится на уровне категории (`SHARED/`)

**Правило**: Если используется в 2+ проектах → общее
